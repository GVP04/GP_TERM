sizing(objnum,CODE)

устанавливает правила масштабирования объекта

Функция sizing определяет, что будет происходить с объектом
при изменении размеров окна.

Параметры

CODE4-х буквенный код, определяющий изменение координат и размеров
объекта относительно базового объекта (base.objnum)
base.objnumбазовый объект (номер объекта). Можно не указывать. По умолчанию
это родительский объект (window, frame, winbox или tabbox).

Порядок следования букв CODE

1-я буквасмещение по горизонтали левого верхнего угла объекта
2-я буквасмещение по вертикали левого верхнего угла объекта
3-я буквасмещение по горизонтали правого нижнего угла объекта
4-я буквасмещение по вертикали правого нижнего угла объекта

Коды букв CODE

Sсохранять расстояние относительно левого верхнего угла базового объекта
Eсохранять расстояние относительно правого нижнего угла базового объекта
Mсохранять расстояние относительно центра базового объекта
Pпропорционально изменять расстояние
Nне применять никаких действий

Примеры Sizing

SSSSОбъект не меняет никаких координат, т.к. сохраняются все
расстояния относительно левого верхнего угла базового объекта.
EEEEШирина и высота объекта не меняются, объект следует за
правым нижним углом базового объекта.
SSEEЛевый верхний угол объекта остается на месте, а правый нижний
сохраняет растояние относительно правого нижнего угла базового
объекта. Ширина и высота объекта увеличивается на столько же,
на сколько увеличиваются высота и ширина базового объекта.
SSPPЛевый верхний угол объекта остается на месте, а правый нижний
не сохраняет растояния относительно правого нижнего угла
базового объекта. Ширина и высота объекта увеличиваются
пропорционально ширине и высоте базового объекта.
SSEMЛевый верхний угол объекта остается на месте, а правый нижний
сохраняет растояние по горизонтали относительно нижнего угла
базового объекта, и по вертикали относительно центра базового
объекта. Ширина и высота объекта увеличиваются.

autowaitcursor(onoff)

режим автоблокировки окна по любому событию

Параметры

onoffфлаг:
1 - режим включен
0 - режим выключен

Этот режим нужен, если сервер сильно нагружен,
и возможны большие задержки в обработке событий.

В этом режиме при создании объектов события в них
будут программироваться таким образом, что перед
отправкой сообщения будет блокироваться окно, а
курсор меняться на песочные часы.

Это необходимо для уведомления пользователя о простое
и предотвращения выполнения им каких-либо действий.

Функция getmsg автоматически разблокирует окно после
обработки события.

Внимание

  • Функция initgui устанавливает этот флаг в 0.
  • Целесообразно отключить в объектах необрабатываемые
    сообщения (с помощью функции unbind)
